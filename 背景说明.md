# 项目背景：北京市智能体引领发展若干措施

> 整理自 `为什么要创建这个网站.txt`，作为后续任务背景参考。

## 一、政策定位

《北京市关于加快智能体引领发展的若干措施》是北京市推动 **Agentic AI / 智能体** 技术创新与产业发展的纲领性文件，共 10 条措施，目标是：

1. 率先培育 **智能经济新形态**
2. 加快建设 **全球人工智能创新高地**

---

## 二、三大核心目标

| 目标 | 要点 |
|------|------|
| 推动智能体创新引领发展 | 将 Agentic AI 视为 AI 发展新阶段，从技术、应用、产业、生态多维度系统布局 |
| 培育智能经济新形态 | 从“消耗 Token 计费”转向“按价值计费”，催生 OPC（一人公司）等新模式 |
| 建设全球 AI 创新高地 | 通过政策引导和要素保障，巩固北京在全球 AI 领域的领先地位 |

---

## 三、十大重点措施

| 序号 | 措施 | 关键内容 |
|------|------|----------|
| 1 | 提升基础模型能力 | 推动 Agentic AI 创新，提升模型完成实际任务的能力；支持在线学习、持续学习、自主进化、超长程任务、复杂推理与规划、工具调用、长上下文、多智能体协同、记忆等 |
| 2 | 强化智能体底层技术 | 首次提出支持 **驾驭层工程（Harness Engineering）**，夯实共性技术底座；重点：上下文工程、任务持久化、多智能体协作、系统可扩展性 |
| 3 | 加速原生应用与标杆场景 | 开发“需求智能”驱动的原生 AI 软件/超级软件；支持 **前沿部署工程师（FDE）** 角色；标杆场景覆盖科学、医疗、教育、政务、制造、文化 |
| 4 | 推动智能终端融合 | 智能体嵌入手机、眼镜、可穿戴设备、机器人、汽车；推动“芯模云端用”五位一体 |
| 5 | 支持创新创业新模式 | 支持 **OPC（一人公司）** 模式，配套弹性算力、孵化、金融、知识产权等全周期服务 |
| 6 | 鼓励发展 Token 经济 | 从 Token 消耗量计费转向价值计费；发展 TaaS、AaaS、RaaS；探索 Token 券和智能体服务券 |
| 7 | 全面提升安全治理能力 | 运行时治理：防范目标理解错误、权限越界、提示词注入；分级监管：自主决策 / 用户授权 / 仅限本人决策 |
| 8 | 加大关键要素保障力 | 实施 **银河算廊工程**；银行和保险开发智能体金融产品；重视“任务轨迹”作为核心数据资产 |
| 9 | 推动开源开放 | 建设中国与上合组织国家 AI 应用合作中心；建设全球影响力开源社区 |
| 10 | 保障措施 | 统筹国家、市区财政资金、政府投资基金和市场化基金支持相关项目 |

## 四、关键概念速查

| 概念 | 含义 |
|------|------|
| **Agentic AI / 智能体** | 具备自主规划、工具调用、长程任务执行能力的人工智能新阶段 |
| **Harness Engineering（驾驭层工程）** | 夯实智能体共性技术底座的工程体系，提升能力下限 |
| **OPC（One Person Company，一人公司）** | 以个人为核心、借助 AI 降低协调成本的新型创业模式 |
| **FDE（Frontline Deployment Engineer，前沿部署工程师）** | 深入客户现场做流程拆解和系统对接，加速产品落地 |
| **Token 经济** | 从按 Token 消耗计费转向按价值计费，衍生 TaaS / AaaS / RaaS 模式 |
| **银河算廊工程** | 面向智能体高频并发、低延迟需求的新型算力基础设施 |
| **任务轨迹** | 智能体运行过程中产生的过程数据，被视为优化模型和业务流程的核心资产 |

## 五、对后续任务的启示

本网站/项目可围绕以下方向展开：

1. **政策解读与知识库**：系统整理北京智能体政策，输出结构化内容。
2. **Agentic AI 应用展示**：结合 Harness Engineering、FDE、OPC 等概念，展示智能体在实际场景中的落地路径。
3. **Token 经济工具/计算**：提供 TaaS / AaaS / RaaS 相关的价值评估或计费工具。
4. **行业标杆场景**：覆盖科学、医疗、教育、政务、制造、文化等垂直领域。
5. **安全与治理**：展示智能体分级监管、权限控制、提示词防护等能力。
